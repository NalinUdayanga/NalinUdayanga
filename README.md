# Professional GitHub Tips

## Keep Repositories Clean

Good structure:

```text
project-name/
 ├── README.md
 ├── scripts/
 ├── docs/
 ├── ansible/
 └── docker/
```

---

## Write Strong README Files

Every repo should contain:

* Project purpose
* Technologies used
* Setup instructions
* Screenshots (if needed)
* Example outputs

---

## Commit Message Examples

Good:

```bash
feat: add ansible playbook for nginx deployment
fix: resolve memory leak issue in monitoring script
docs: update installation guide
```

Bad:

```bash
update
fix
changes
```
